# SpeedyWatch Project DOX

## Purpose and Ownership

- This project owns the SpeedyWatch Android and native iPhone apps: YouTube WebView browsers with native playback-rate controls, Android MP3/MP4 downloads, searchable caption transcripts, OpenRouter video summaries with follow-up chat, savable pre-watch question guides, and best-effort DOM ad skipping.
- Keep source, generated app artifacts, and verification evidence inside this project boundary. Do not reuse release artifacts or private configuration from sibling projects.
- `README.md` and `logo.png` are the public GitHub landing assets. Keep their feature claims, installation requirements, release metadata, and privacy disclosures aligned with the shipped app.

## Product Contracts

- Support Android 10 and newer with package name `com.speedywatch.app`.
- Load YouTube only over HTTPS in the app WebView. YouTube navigation remains in-app; unrelated links open through the current platform's external handler.
- Keep WebView file access, content access, mixed content, popup windows, and WebView debugging disabled. Do not add a JavaScript bridge.
- `app/src/main/assets/speedywatch.js` is the single controller for playback speed, ad skipping, caption-track discovery, caption-request triggering, and timestamp seeking. Native code may call only its fixed methods with bounded numeric or boolean values and must not add a JavaScript bridge.
- Playback speed is bounded to 0.25x through 4x. The native controls provide common presets, 0.1x adjustments, and direct decimal entry; Settings persists a custom default rate that initializes future app launches and applies immediately when saved. The controller restores the chosen rate when YouTube replaces or resets media elements.
- While `MainActivity` is foregrounded, keep its window awake with Android's `FLAG_KEEP_SCREEN_ON`; do not poll, request a wake-lock permission, or keep the display awake after the app leaves the foreground.
- Ad skipping defaults to on. It removes known feed-ad DOM nodes and uses five ordered strategies derived from the local GMB blocker: `cancelPlayback()` plus delayed `playVideo()`, visible skip-button clicks, media seek-to-end, player `skipAd()`, and ad-slot parent removal. Hidden skip buttons must not short-circuit later fallbacks. Detection remains best effort because YouTube DOM and private player methods can change; this is not a network-level blocker.
- The compact top toolbar uses icon-only home, history, reload, YouTube Subs, Quiz, Download, Saved, and Settings actions. Download sits immediately left of Saved, Saved uses a thin bookmark icon immediately left of Settings, and Settings remains the rightmost action. Do not restore the removed copy-URL or app-fullscreen toolbar actions unless the product requirement changes.
- Android downloads are self-contained in the APK and offer MP3 audio plus available MP4 resolutions for the active validated HTTPS YouTube video. The app bundles the official yt-dlp `2026.07.04` executable and youtubedl-android/FFmpeg runtime, replaces an older private executable when the bundled version changes, performs work in a cancellable foreground data-sync service, and publishes completed files through MediaStore under `Downloads/SpeedyWatch`. Standard choices remain available if metadata discovery fails, and MP4 selection is an upper resolution bound with H.264/AAC preferred for Android compatibility. Do not expose implementation-component names in app UI or download notifications. Downloads remain best effort for publicly available media because YouTube delivery interfaces and access restrictions can change; iPhone behavior is unchanged.
- Completed Android media filenames derive from the title reported by the actual yt-dlp download, not from dialog timing or placeholder metadata. Remove characters unsafe for Android display names, bound the display-name length, retain the MP3 or MP4 extension, and let MediaStore resolve duplicate names.
- Near the bottom of Settings and README, show `Brought to you by the team from ` as regular text followed by the linked, capitalized brand text `SEO Time Machines` pointing to `https://seotimemachines.com`. In Settings, the regular text remains white, only the brand text is link-styled and tappable, and the link opens through the platform's external HTTPS handler rather than the app WebView.
- The YouTube Subs modal loads timedtext JSON, filters captions case-insensitively, seeks through the fixed controller method, and closes after a timestamp selection. Summary results render common Markdown as readable native styled text. Every successful Summary One or Summary Two result exposes Copy summary, Save summary, and Share summary actions plus a bottom follow-up composer under the visible `Continue with a question` heading; the success status directs the user to ask below. Sharing uses the platform-native plain-text surface (`ACTION_SEND` chooser on Android and `ShareLink` sheet on iPhone) and includes the original validated HTTPS YouTube URL. Follow-up chat preserves the current modal's conversation while saved, copied, and shared content remains the original summary. Save summary persists the raw Markdown, summary type, video title, original HTTPS YouTube URL, and save timestamp in app-private storage (`saved_summaries.db` SQLite on Android and an atomically written JSON collection on iPhone). Prefer the signed track exposed by the page controller; when the mobile page omits it, use the no-key Android InnerTube client profile as an unofficial fallback and accept only HTTPS YouTube `/api/timedtext` URLs.
- OpenRouter settings dynamically load text-output models from `https://openrouter.ai/api/v1/models`, prefer `inception/mercury-2` when available, and keep Summary One, Summary Two, and Quiz prompts editable. AI requests use the exact saved field prompt as the system instruction. Initial user messages contain only neutral source metadata and transcript data, with the selected Quiz count included as request metadata. Summary follow-ups keep that originating prompt unchanged, preserve prior user and assistant turns for the current modal, and add only the user's explicitly labeled question. Android resources and iPhone `AppSettings` own matching platform defaults; Summary One defaults to the detailed business implementation prompt, and an exact previously shipped stock Summary One value migrates to the current default without replacing user-edited text. Android generation also resolves a missing Summary One value to that resource default. Empty user-entered prompt fields are rejected. Do not add hidden output, structure, timestamp, quiz, or chat instructions to request builders. The API key field defaults to masked, exposes a short prefix/suffix check, and has an explicit visibility toggle. Persist the key only in platform-protected storage: Android Keystore AES-GCM ciphertext or iPhone Keychain. A DEBUG-only iPhone process environment key may support local tests, but keys must never be logged, exported, committed, or bundled.
- The Quiz action sends the active video's transcript and the selected 6, 10, 12, or 20 question count as neutral request data. The editable Quiz prompt solely controls the generated output. Save quiz and Share quiz become available after successful generation. Saving persists the raw output with a question-count label, video title, original HTTPS YouTube URL, and save timestamp in the same app-private saved-content store as summaries. Sharing uses the platform-native plain-text surface and includes that original validated URL.
- The Saved modal lists summaries and quizzes newest first, searches case-insensitively across video titles, content types, headings, and full saved text, and opens a Markdown-rendered detail view that shows the original URL. The detail view also shares the saved content and original URL through the platform-native share surface. Opening or sharing the source must validate an HTTPS YouTube host. Opening loads it in the existing WebView. Deletion requires confirmation.
- Android modal layouts keep at least 8dp of horizontal or vertical separation between adjacent text, buttons, and controls while preserving touch targets of at least 44dp.
- Android Settings checks the official `demetre19/SpeedyWatch` latest stable GitHub Release at most once per day when opened and provides explicit Check and Download buttons. Accept only numeric `vX.Y` through `vX.Y.Z.W` tags, the exact official release and `SpeedyWatch.apk` download paths, APK MIME type, bounded positive size, and GitHub SHA-256 metadata. Compare version components numerically, warn before downloading a published build that is not newer, and enqueue accepted downloads into the public Downloads folder without silently installing them.
- The Android launcher icon is an adaptive dark/red speed, play, and watch mark with a legacy fallback; the iPhone asset catalog uses the matching branded app icon.
- Public APK delivery uses GitHub Releases in `demetre19/SpeedyWatch`. Each release attaches the exact device-verified APK as `SpeedyWatch.apk`; the README's stable latest-download link must remain `https://github.com/demetre19/SpeedyWatch/releases/latest/download/SpeedyWatch.apk`.
- In this project, a user request to publish or push a completed, tested Android version to the repository includes committing and pushing its source changes, creating the matching stable GitHub Release, attaching the exact device-verified APK as `SpeedyWatch.apk`, and verifying the stable latest-download URL against the recorded SHA-256. Treat it as source-only only when the user explicitly says not to create or update a release.
- Until a production signing workflow is added, public APKs are debug-signed and the README and release notes must say so. Do not imply production signing or seamless signer migration.
- Public iPhone delivery is source-only in the separate `ios/` folder until an Apple-signed IPA, TestFlight, or App Store workflow exists. README download choices must distinguish the installable Android APK from the iPhone source ZIP/Xcode project, state that physical iPhone installation requires developer signing, and never imply that the source archive is directly installable.

## Build and Verification

- Use JDK 17 and the checked-in Gradle 9.4.1 wrapper. On this workstation set `JAVA_HOME=/Volumes/TheHoneyBadger/AndroidTooling/jdks/jdk-17.0.19+10/Contents/Home`, `ANDROID_HOME=/Volumes/TheHoneyBadger/AndroidTooling/android-sdk`, and `ANDROID_SDK_ROOT=$ANDROID_HOME`.
- Build the debug APK with `./gradlew --no-daemon :app:assembleDebug`.
- Install with `./gradlew --no-daemon :app:installDebug` while an emulator or phone is connected.
- Before publishing, calculate the exact APK SHA-256, update the README metadata, upload that same device-verified file without rebuilding as the GitHub Release asset, and verify the public latest-download URL resolves to byte-identical content.
- The existing `cmuxPhoneApi36` AVD is the primary local smoke-test profile. Verify that YouTube renders, direct 2.7x entry and 0.1x stepping report matching live media rates, the ad-skipping toggle changes controller state, Settings loads the live OpenRouter catalog with Mercury 2 selected and exposes the masked API-key check and all three editable prompt fields, a captioned `/watch` video supports transcript search and timestamp seek with modal dismissal, Summary One and Summary Two each follow only their saved prompt, successful summaries expose Copy, Save, and Share actions plus continued question chat, Saved persists summaries and quizzes across modal/app reopen with source URLs, heading/body/type search filters correctly, source URLs reopen in the WebView, saved details share through the platform-native surface with the original URL in the payload, and Quiz follows only its saved prompt while receiving the selected count as request metadata and exposing Save and Share actions after generation.
- For Android download changes, verify on `cmuxPhoneApi36` that the shipping dialog always exposes MP3 and standard MP4 choices while metadata loads, replaces them with discovered resolutions when available, and completes one MP3 plus one H.264/AAC MP4 through the real foreground service into `Downloads/SpeedyWatch`. Also verify a stale private yt-dlp executable is replaced by the bundled version without clearing app data.
- For Android filename changes, start a download while the dialog still shows `Checking this video...` and verify the completed MediaStore display name uses the YouTube title rather than `YouTube Video`.
- Run `node --check app/src/main/assets/speedywatch.js` after changing the shared injected controller, then build and launch the affected platform for behavioral changes.
- For iPhone builds, live UI parity tests, and secure local OpenRouter injection, follow `ios/AGENTS.md`.
- Every meaningful contract, structure, permission, toolchain, or verification change requires a DOX pass in this file and, when the child boundary changes, the workspace parent index.
